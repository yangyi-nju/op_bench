# OpBench v0.6 Real MCP Full-Agent Experiment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> `superpowers:subagent-driven-development` (recommended) or
> `superpowers:executing-plans` to implement this plan task-by-task. Use
> `superpowers:test-driven-development` for every production change,
> `superpowers:systematic-debugging` for every unexpected failure, and
> `superpowers:verification-before-completion` before reporting any gate or
> experiment complete. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an attempt-scoped real stdio MCP transport for the canonical v0.6
Action path, prove it with offline conformance plus local and exact-target
canaries, and then run the frozen 17-task Dataset three times through real Codex
for 51 selected Attempts across four distinct Runtime Profile cohorts.

**Architecture:** `Codex CLI` launches one stdlib-only MCP server in its exact
process group. The MCP server invokes the existing descriptor-bound
`opbench_action.py`; `ProcessActionExchange` forwards requests to the sole
controller-side `CanonicalActionService`. MCP owns protocol negotiation and
tool presentation only. Action validation, Budget, Workspace mutation, Event
Trace, Patch Freeze, Runtime, Fresh Evaluation, resume, and Integrity remain in
their current authoritative components. A public adapter trace records only
sanitized protocol counters and identities; private argv, environment, process
handles, target binding, and credentials never enter public artifacts.

**Tech Stack:** Python 3.11 standard library, JSON-RPC 2.0 over line-delimited
stdio, MCP protocol versions `2024-11-05`, `2025-03-26`, and `2025-06-18`,
Codex CLI `0.145.0-alpha.18` (re-detected and frozen immediately before the
canary), model `gpt-5.6-sol`, existing OpBench v0.6 canonical contracts,
`unittest`, local Git, exact subprocess argv, Docker CLI, and SSH/rsync only to
the one target already present in `configs/remote_hosts.json`.

## Global Constraints

- The approved design is
  `docs/superpowers/specs/2026-07-20-opbench-v0.6-real-mcp-full-agent-experiment-design.md`.
  If implementation pressure conflicts with that document, stop and amend the
  design before changing behavior.
- Preserve `codex_canonical` as a compatibility Adapter. The formal experiment
  uses only the new independent `codex_mcp_canonical` identity.
- Preserve all 14 existing Integrity check IDs. Bind MCP metadata under the
  existing `lifecycle_terminal` graph; do not silently change the release
  denominator from 14.
- Never execute ping, port scan, service scan, host/address discovery, alternate
  target search, process/container enumeration, or name/filter-based cleanup.
  Runtime operations address only the exact configured target and exact
  descriptor-owned resources.
- Normal controller-to-Codex Provider traffic is allowed. Task data-plane
  networking remains the selected Runtime Profile's `network_policy=denied`.
- Do not write Codex global configuration. MCP is injected only by invocation-
  local `codex exec -c ...` overrides, with `--ephemeral`,
  `--ignore-user-config`, and explicit `--model gpt-5.6-sol`.
- Put Codex, its MCP server, and descendants in one recorded process group.
  Cleanup may signal only that exact group ID, first `SIGTERM` and then
  `SIGKILL` after the fixed grace period. Do not use `pkill`, `pgrep`, `ps`, or
  process-name matching.
- `ActionObservation.ok == false` is a normal tool result and Agent trace fact.
  It is not an MCP or infrastructure failure.
- A valid Agent `timeout`, `missing_finish`, `no_patch`, or `f2p_failed` result is
  final and is not retried. Only missing or infrastructure-invalid logical
  Attempts receive a new retry index.
- Do not remove or stage unrelated untracked `runs/v0.6_retry_*`, release private
  evidence, or historical replay roots. Never use `--fresh` for v1 runs.
- The four formal roots are immutable and must be absent before their first
  invocation. If any already exists, inspect its manifest and resume only when
  the full identity matches; never delete it to force a restart.
- Run the v1 runner serially (`--max-parallel 1`, its default). CUDA cohorts are
  never concurrent.
- Every production behavior follows RED → minimum GREEN → focused regression.
  Commit the implementation platform before any formal cohort so the experiment
  source identity cannot drift mid-run.



  ```text

  invalid request           -> -32600
  method not found          -> -32601
  invalid params/tool       -> -32602
  internal action bridge    -> -32603
  ```

  An unsupported requested MCP version returns server version `2025-06-18` in
  the initialize result. A normal Action observation with `ok=false` remains a
  successful `tools/call` result.

- Trace writes use an adapter-private path in scratch and an atomic regular-file
  replace. The file contains only counts, negotiated version, and terminal
  status; argv, paths, environment, JSON-RPC payloads, and observations are not
  recorded there.
