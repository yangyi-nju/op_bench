# OpBench v0.6 Controller-Side Evaluation Staging Plan

**Goal:** Remove the undeclared in-container Git dependency while preserving
strict source and patch semantics, then produce corrected exact replay evidence.

**Design:** `docs/superpowers/specs/2026-07-18-opbench-v0.6-controller-side-evaluation-staging-design.md`

## Task 1: Normalize legacy executable revisions and container Git authority

- Add a failing test that all 17 private v0.5 bindings select a commit present
  in their immutable snapshot repository.
- Add failing Docker and Remote command tests for exact
  `safe.directory=<workspace_target>` command-level configuration.
- Select the logical base commit when present; otherwise explicitly select the
  snapshot repository's concrete HEAD commit.
- Inject exact, non-wildcard Git protected configuration into container exec.
- Run legacy, backend, replay, and Runtime evaluation focused suites.

## Task 2: Move source verification and strict patch staging to the controller

- Add a failing Runtime evaluation test that a gold replay resolves without any
  Runtime command invoking Git or the old in-container source-hash program.
- Resolve and validate the exact workspace resource owned by the current lease.
- Verify the frozen commit and `content_sha256` archive identity locally.
- Apply Agent and hidden patches with local `git apply --check` and strict apply
  under isolated Git configuration.
- Leave runner-file injection, source loading, and selectors in the Runtime.
- Run the direct Runtime evaluation and backend suites.

## Task 3: Verify local release scope

- Run all `test_runtime_*.py` tests.
- Run the complete `test_*.py` suite.
- Validate Dataset, example RunManifest, tracked JSON, and `git diff --check`.
- Reconfirm the polluted source cache remains untouched.
- Commit the implementation milestone.

## Task 4: Gate and rerun exact replay

- Execute one exact baseline case for
  `pytorch__124385__load_state_dict_prefix`; require observed `f2p_failed`,
  matching the frozen expectation, and complete cleanup.
- Recompute all eight historical v0.5 hashes.
- Run a new complete replay under
  `runs/v0.6_retry_legacy_replay_exact_fixed/`; do not overwrite failed
  evidence.
- Require 85 unique results, expected 17+17+51 case counts, zero Blocked/Failed,
  explicit difference attribution, and unchanged historical hashes.
- Verify every retained Runtime resource ledger or document the implementation
  gap truthfully if replay does not persist private ledgers.

## Task 5: Close release documentation

- Record both failed and corrected replay evidence and all canary evidence.
- Update Must rows only from direct artifacts.
- Run the final release audit and full verification.
- Commit the documentation/release milestone.
