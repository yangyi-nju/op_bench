# OpBench v0.6 M4 Session, Events, Budget, and Resume Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add one deterministic AttemptSession lifecycle, append-only public
trajectory, server-owned budget/termination convergence, and v0.6 resume ledger.

**Architecture:** `AttemptSession` owns state and terminal arbitration,
`CanonicalActionService` remains the only Action authority, and both write to one
injected `EventJournal`. `PublicArtifactStore` spills large public payloads while
the Journal keeps canonical references. `AttemptLedger` is an independent,
append-only resume index keyed by the already-frozen M1 Attempt Identity.

**Tech Stack:** Python standard library, immutable v0.6 contracts, canonical JSON
and SHA-256, descriptor-safe M2 Workspace, `unittest`, local Git fixtures.

## Global Constraints

- Do not run an Agent, Docker, SSH, remote runtime, network scan, or network probe.
- Preserve the M1-M3 public contracts and v0.5 default execution path.
- Every public trajectory value must pass the existing public-artifact scanner.
- Event sequence and hash assignment are server-owned and append-only.
- The event hash preimage is the complete EventRecord envelope except
  `event_hash`; it includes `previous_event_hash`.
- Terminal candidates admitted before `finalize()` are ranked, highest first:
  `platform_error`, `workspace_error`, `runtime_error`, `provider_error`,
  `cancelled`, `timeout`, `budget_exhausted`, `agent_finished`, `agent_exited`.
- `finalize()` is the arbitration seal; later stop signals cannot alter the result.
- Infrastructure terminal reasons are never classified as an Agent repair failure.
- Use TDD for every production behavior: write the test, observe the intended
  failure, implement the minimum, then rerun focused compatibility tests.

- [x] **Step 3: Implement lifecycle and Action bridge**

  service; it requests `agent_finished`, `budget_exhausted`, or `timeout` from the

- [x] **Step 4: Write and run the failing terminal-race matrix**

  Use barriers to submit every pair of stop reasons before one `finalize()` call.
  Assert the priority in Global Constraints, exactly one freeze invocation, one
  `terminal_emitted` Event, one immutable `SessionResult`, and identical return
  objects for concurrent/repeated finalizers.

- [x] **Step 5: Implement finalize/freeze convergence**

  `finalize()` seals candidates, transitions to `freezing`, emits freeze start,
  invokes `freeze_patch`, and emits completed/failed. Freeze failure admits
  `workspace_error` before final arbitration. The result uses the selected reason,
  exact frozen patch identity (or null on freeze failure), and deterministic
  session timestamps. Requests after the seal are audit no-ops and cannot change
  terminal state, patch, or event tail.

  `SessionSpec.deadline_ms` and service-owned wall/action/test/command/output
  usage; request timeout or budget stop without re-counting usage.

- [x] **Step 7: Run Task 3 plus M2/M3 compatibility tests**

  Require all session/race/action/freeze modules to pass.

### Task 4: Append-only AttemptLedger and deterministic resume

**Files:**

- Create: `src/op_bench/runtime/resume.py`

**Interfaces:**

- Produces: `AttemptLedgerRecord(attempt_id, session_id, retry_index,
  attempt_validity, session_result, session_result_hash, recorded_at_ms)`.
- Produces: `ResumeDecision(action, retry_index, reason)` where action is
  `run`, `skip`, or `blocked`.
- Produces: `AttemptLedger(path).append(...)`, `records(attempt_id=None)`,
  `decide(attempt_id, resume_policy)`, and `latest_valid(attempt_id)`.

- [x] **Step 1: Write failing append/resume tests**

  Require fsynced one-line canonical JSON records, strict field validation,
  monotonic retry indexes, unique session ids, exact duplicate idempotency, and
  conflict on changed duplicate content. A malformed/truncated line must make the
  v0.6 ledger fail closed rather than silently alter resume decisions.

- [x] **Step 2: Define and test resume policy truth table**

  Use this exact table:

  | History | `skip_valid` | `retry_infrastructure` | `never` |
  | --- | --- | --- | --- |
  | none | run #1 | run #1 | run #1 |
  | valid | skip | skip | blocked |
  | infrastructure-invalid only | blocked | run next retry | blocked |

  Repeated `decide` calls are read-only and byte-stable. Once a valid record exists,
  later retry append is rejected. `latest_valid` returns the last valid logical
  result without removing raw retry rows.

- [x] **Step 3: Run resume tests and verify RED**

  Expected: import failure for `op_bench.runtime.resume`.

- [x] **Step 4: Implement strict append-only ledger**

  Reuse `SessionResult.to_dict()` and its `content_hash`; store the complete result,
  not a summary. Append with a bound regular-file descriptor, `O_APPEND`, locks,
  locked tail revalidation, and file/directory `fsync`; parse and validate the
  whole file on every process open. A failed uncommitted transaction may roll back
  only its own bytes; committed rows are never rewritten, truncated, or
  deduplicated.

- [x] **Step 5: Prove Attempt Identity mutation and retry audit**

  Reuse M1 `attempt_id(...)` and add a table-driven test showing Task, Agent,
  Prompt, Capability Policy, Budget, Action Protocol, and Runtime/Profile changes
  yield distinct IDs. Append two infrastructure-invalid retries then one valid
  result and prove the raw ledger has three rows while logical selection has one.

- [x] **Step 6: Run v0.6 and Legacy resume compatibility tests**

  Run `tests.test_runtime_resume`, `tests.test_runtime_manifest`,
  `tests.test_runtime_manifest_cli`, `tests.test_resume`, and
  `tests.test_run_experiment`. The new ledger must not change v0.5 behavior.
