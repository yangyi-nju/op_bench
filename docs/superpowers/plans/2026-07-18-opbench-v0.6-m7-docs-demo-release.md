# OpBench v0.6 M7 Documentation, Demo, and Release Review Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finish the locally executable v0.6 documentation and demo surface, publish reproducible M7 evidence, and make a truthful release decision without hiding the blocked exact-Remote replay/CUDA gates.

**Architecture:** Keep the existing v1 Runtime as the only execution path. A small fixture-preparation CLI materializes a synthetic verified Dataset and deterministic local Git source; the documented smoke, resume, Integrity, resource, and real-Codex commands then call the existing production CLIs. Release documentation records a compact public artifact index and exact evidence, while private runtime handles and raw run roots remain untracked.

**Tech Stack:** Python 3.9+ standard library, `unittest`, Git CLI with fixed author/committer metadata, existing OpBench v1 Runtime CLIs, Markdown, canonical JSON.

## Global Constraints

- v0.6 is one release; M1–M7 are internal checkpoints and do not create separate public versions.
- Preserve the v0.5 default CLI path; v1 remains an explicit `--runtime-protocol v1` selection.
- Do not add a second Demo evaluator, runner, Adapter, or scoring path.
- Do not run network probes, host discovery, service discovery, broad process/container enumeration, or searches for replacement targets.
- Exact configured OpenAI/Codex and remote-target operations are allowed, but the already stable Remote `connection_timeout` is recorded rather than retried during M7.
- Never track private target configuration, exact runtime handles, machine-local paths, credentials, hidden real-task assets, or raw M6 run roots.
- The public Demo is synthetic and must not be presented as a v0.6 benchmark score.
- v0.5 historical results remain v0.5 evidence and must not be relabeled as v0.6.
- The release decision remains `Blocked` while R-05–R-08 or R-10 are not `Passed`, even when M7 itself is complete.
- Use `apply_patch` for repository edits and keep the current `codex/opbench-v0.6` branch selected per the recorded project decision.
