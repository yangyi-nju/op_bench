# OpBench v0.6 M5 Fresh Evaluation, Artifact, Integrity, and Summary Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Evaluate each frozen patch in a fresh source copy, preserve isolated
public/private evidence, classify results on three independent axes, and rebuild
verified v0.6 results and summaries from raw attempt artifacts.

**Architecture:** `FreshEvaluator` is a backend-neutral classifier that accepts
only immutable Session/Evaluation/Patch identities. A local Git backend proves
fresh-copy, strict-apply, hidden-test injection, and F2P/P2P semantics without
reading the Agent workspace. `AttemptArtifactStore`, `AttemptLedger`, the v0.6
summary rebuilder, and the integrity verifier form one canonical artifact graph;
the legacy `op_bench.evaluator` and `op_bench.reporter` remain unchanged until
M6 compatibility adapters are added.

**Tech Stack:** Python standard library, immutable v0.6 contracts, canonical JSON
and SHA-256, local Git and `unittest` fixtures, append-only EventJournal/Ledger.

## Global Constraints

- Do not run an Agent, Docker, SSH, remote runtime, network scan, or network probe.
- Preserve the v0.5 default evaluator, reporter, result, and CLI behavior.
- Use a fresh local source repository fixture; never pass an Agent workspace path
  into `FreshEvaluator` or `LocalGitEvaluationBackend`.
- Apply the frozen Agent patch strictly with `git apply --check` followed by
  `git apply`; do not use `patch`, fuzz, offset fallback, or three-way merge.
- Inject the evaluation-only test asset only after Session terminal evidence and
  after the frozen Agent patch has been applied.
- A selector with zero collected tests, zero executed tests, or all tests skipped
  is evaluator invalidity, never a passing test result.
- `attempt_validity`, `agent_terminal`, and `evaluation_outcome` are independent;
  infrastructure-invalid rows stay in raw results but never enter the resolved
  denominator.
- Agent Session finalization emits `session_terminal_emitted`; only the completed
  Attempt emits the unique final `terminal_emitted` event.
- Public artifacts must pass the existing public-artifact scanner and contain no
  hidden source, gold content, private test output, credential, or host path.
- v0.6 is unreleased and M1-M7 are internal implementation milestones. The
  checked-in `v1` contracts describe the first v0.6 release, so pre-release M5
  closure may update them in place; any breaking change after v0.6 release must
  use a new schema version and an explicit migration policy.
- Use TDD for every production behavior: write one focused test, observe the
  intended failure, implement the minimum, then rerun focused compatibility tests.
