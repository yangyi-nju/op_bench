# OpBench v0.6 Revision-Only Source Materialization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make every executable v0.6 Runtime workspace consume only the exact frozen Git revision, then use the corrected path to close the outstanding exact Remote CPU/CUDA and 17+17+51 replay gates.

**Architecture:** Add an explicit private source revision to `RuntimeAttemptContext`. A focused source-materialization module streams the selected local Git tree into an Attempt-owned directory, validates archive paths and links, creates a deterministic standalone Git snapshot, and proves its tree equals `<revision>^{tree}`. Local, Docker, and Remote Docker reuse that helper; public identities and schemas stay unchanged.

**Tech Stack:** Python 3.11+ standard library, `dataclasses`, `pathlib`, `subprocess`, `tarfile`, local Git CLI, existing `unittest` suite, existing v0.6 Runtime resource/Integrity verifiers.

## Global Constraints

- Materialize from the exact frozen Git revision, never mutable cache working-tree state.
- Do not checkout, clean, reset, register a worktree in, or remove files from the shared source cache.
- Preserve public Runtime contract/schema version `v1` and all existing Dataset, Task, Source, Environment, Profile, Cohort, Attempt, scoring, and v0.5 historical identities.
- No ping, host/port/service discovery, address discovery, process/container enumeration, broad cleanup, fallback target search, or wildcard cleanup.
- Any external run may address only `configs/remote_hosts.json` and must retain existing SSH host identity verification.
- Target values, raw handles, private paths, and private exception messages must not enter public artifacts.
- Every production change follows red-green-refactor and every milestone ends with exact tests and an intentional commit.
- Do not mark R-05, R-06, R-07, R-08, or R-10 Passed without direct exact-runtime artifacts.

```python
            root = Path(temporary)
            source = root / "source"
            revision = initialize_git_repo(source)
```

```python
@dataclass(frozen=True)
class FrozenSourceSnapshot:
    workspace: Path
    revision: str
    source_tree: str
    snapshot_commit: str


class SourceMaterializationError(Exception):
    pass


def materialize_frozen_git_revision(
    source_directory: Path,
    revision: str,
    workspace: Path,
) -> FrozenSourceSnapshot:
    """Create one clean standalone Git workspace from an exact local revision."""
```

Implementation sequence:

```python
resolved_commit = _git_text(source_directory, "rev-parse", "--verify", f"{revision}^{{commit}}")
source_tree = _git_text(source_directory, "rev-parse", "--verify", f"{resolved_commit}^{{tree}}")
workspace.mkdir(parents=True, exist_ok=False)
_stream_safe_git_archive(source_directory, resolved_commit, workspace)
_git(workspace, "init", "--quiet", "--initial-branch=main")
_git(workspace, "add", "--force", "--all")
snapshot_tree = _git_text(workspace, "write-tree")
if snapshot_tree != source_tree:
    raise SourceMaterializationError("materialized Git tree does not match frozen revision")
snapshot_commit = _deterministic_root_commit(workspace, snapshot_tree)
_git(workspace, "update-ref", "HEAD", snapshot_commit)
if _git_bytes(
    workspace,
    "status",
    "--porcelain=v1",
    "-z",
    "--untracked-files=all",
    "--ignored=matching",
):
    raise SourceMaterializationError("materialized workspace is not clean")
return FrozenSourceSnapshot(workspace, resolved_commit, source_tree, snapshot_commit)
```

Use fixed author/committer name, email, timestamp, message, `GIT_CONFIG_NOSYSTEM=1`, `GIT_CONFIG_GLOBAL=os.devnull`, `GIT_ATTR_NOSYSTEM=1`, `GIT_NO_REPLACE_OBJECTS=1`, and `core.autocrlf=false`. The stream extractor must validate each member before extraction and reject all types except directory, regular file, and lexically-contained relative symlink. On any exception, remove only `workspace` and raise `SourceMaterializationError` without printing it.
