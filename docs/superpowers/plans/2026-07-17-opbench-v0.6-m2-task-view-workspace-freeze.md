# OpBench v0.6 M2 AgentTaskView / Workspace / Freeze Implementation Plan

> **Status:** Complete (2026-07-17)
>
> **Milestone scope:** Acceptance T-01 through T-07 and W-01 through W-10.
> Network activity is out of scope. Tests use temporary local Git repositories only.

## Goal

Build the public task projection and the single authoritative workspace boundary that
all later v0.6 Actions, tests, diffs, session results, and evaluations can share. A
freeze must produce one immutable, policy-checked patch identity that strictly
applies to a clean copy of the declared base.

## Design boundaries

- `FullTaskSpec` remains in the control/evaluation plane.
- Adapters consume an `AgentLaunchInput` that can only contain
  `AgentTaskView` and its `task_view` identity.
- Projection copies an explicit field whitelist and then scans every public string.
- Public data rejects direct answer sources, credentials, local absolute paths, and
  hidden/gold/admission/private-output markers.
- `AuthoritativeWorkspace` is the only mutable authority. Its identity is derived
  from base source identity, base commit, materialization mode, and policy — never
  from a machine-local path.
- Workspace paths are normalized POSIX-relative paths. Traversal, absolute paths,
  symlink components, special files, illegal modes, binary writes, and out-of-scope
  writes fail closed.
- Freeze changes state from `open` to `freezing`, rejects new mutations, waits for
  in-flight mutations, captures patch-scope files through the bound root descriptor,
  generates one canonical patch in an isolated Git stage, verifies a strict
  commit-base apply, then transitions to `frozen`.
- Only paths in patch scope enter the frozen patch. Untracked caches and agent test
  changes outside that scope never reach evaluation.
