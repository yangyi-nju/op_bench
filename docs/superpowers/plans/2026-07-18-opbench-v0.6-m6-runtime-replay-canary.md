# OpBench v0.6 M6 Runtime, Replay, and Canary Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> `superpowers:subagent-driven-development` or `superpowers:executing-plans` to
> implement this plan task-by-task. Use `superpowers:test-driven-development`
> for every production change and `superpowers:verification-before-completion`
> before the M6 milestone is reported complete. Steps use checkbox (`- [ ]`)
> syntax for tracking.

**Goal:** Deliver the independent v0.6 Runtime Spine, prove exact
Attempt-owned resource lifecycle semantics, replay the frozen v0.5
17 Baseline + 17 Gold + 51 Legacy Patch matrix without rewriting history, and
complete the required real Codex CPU canary and small batch through the standard
Adapter/Action path.

**Architecture:** A complete `RuntimeProfile` freezes mount, resource, cleanup,
hardware, network, source-loading, image, and timeout semantics. Each retry owns
an append-only public resource ledger plus a descriptor-bound private lease store.
Local process, Docker, and explicit-target Remote Docker backends implement one
prepare/run/collect/cleanup protocol and never discover global resources. The
v0.6 orchestrator binds those backends to the existing Authoritative Workspace,
Canonical Action Service, AttemptSession, FreshEvaluator, Artifact Store,
Integrity verifier, resume ledger, and summary rebuilder. Conformance and Replay
use the same components; they do not introduce alternate action or scoring rules.

**Tech Stack:** Python 3.11 standard library, immutable v0.6 contracts and JSON
Schema, canonical JSON/SHA-256, descriptor-bound append-only artifacts, local Git,
subprocess argv execution, Docker CLI, SSH/rsync only for one explicitly supplied
private target, and `unittest` fixtures.

## Global Constraints

- Preserve the legacy behavior of `scripts/run_experiment.py` unless the caller
  explicitly passes `--runtime-protocol v1` and `--runtime-profile <profile-id>`.
- Do not add or execute ping, port scan, host discovery, address discovery,
  container enumeration, process enumeration, broad identity probing, or fallback
  target search. A Remote operation may address only the exact private target
  supplied by the caller.
- Do not disable or weaken ordinary SSH host identity verification. Use the
  caller's existing SSH configuration and exact target; a direct operation failure
  becomes stable `Blocked` or `platform_error` evidence.
- Provider access is permitted only through the existing Codex CLI path and only
  when the caller explicitly enables the external canary. Runtime code does not
  call a Provider API itself.
- Task data-plane network follows `RuntimeProfile.network_policy` and defaults to
  `denied`. Controller-to-Provider traffic is separate from task data-plane
  traffic.
- Never publish a credential, hostname, remote username, identity-file path,
  machine-local absolute path, PID, container raw name, or remote workspace raw
  path. Public artifacts contain logical identities and raw-handle hashes only.
- Never clean up a resource discovered by a filter or global listing. Cleanup
  consumes only the current retry's descriptor-bound private lease store and
  operates on each exact handle recorded there.
- A cleanup failure preserves the original Session/Evaluation attribution but
  fails Integrity with `cleanup_failed`. It never becomes a successful Attempt.
- Replay is compatibility evidence. It never creates an Agent Session, enters a
  v0.6 Agent cohort, modifies v0.5 `results.jsonl`/`summary.json`, or counts an old
  patch as a new Agent score.
- Real Codex R-09 and R-11 must be `Passed`. Remote/CUDA R-10 may be `Blocked`
  only with an exact profile, exact target/config availability reason, and no
  inferred success.
- Use TDD for every production behavior: add one focused failing test, run it and
  record the intended RED, implement the minimum behavior, then run the focused
  compatibility set to GREEN.
- The user requested one commit per completed milestone. Do not create partial
  implementation commits inside M6; commit the approved design separately
  (already complete) and commit the complete M6 implementation once all mandatory
  M6 gates pass.
