# OpBench v0.6 Controller-Side Evaluation Staging Design

Date: 2026-07-18

Status: Approved by the user's standing instruction to proceed without further
approval pauses.

## Context

The corrected exact Remote canaries proved that CPU, CUDA Overlay, and CUDA
Kernel containers can be created and cleaned safely. The first exact 17+17+51
replay then exposed two compatibility facts that no-patch canaries do not
exercise:

1. Several immutable v0.5 source caches contain a clean synthetic snapshot
   commit but not the logical upstream base commit object.
2. The frozen Runtime images contain Python and PyTorch but do not contain the
   Git executable. `RuntimeFreshEvaluationBackend` currently invokes `git
   archive` and `git apply` inside those containers, so every case that reaches
   evaluation fails before its hidden tests.

Rebuilding the images would change their frozen identities. Depending on Git
on the Remote host would move evaluation staging outside the declared Runtime
Profile and add an undeclared host dependency.

## Decision

Evaluation staging will use the Attempt-owned local snapshot that every
executable backend already creates:

1. Resolve the exact workspace resource from the current Runtime lease and
   validate that it is the real Attempt-owned workspace under the configured
   local workspace parent.
2. Verify the workspace's explicit frozen commit locally. For
   `content_sha256` Source identities, recompute the canonical `git archive`
   identity locally and require equality. For legacy `canonical_config`
   identities, require the exact executable snapshot commit selected by the
   private binding.
3. Apply the frozen Agent patch and evaluation-only hidden patch locally with
   controller Git, using `git apply --check` followed by strict apply, with
   system/global configuration disabled and no network operation.
4. Keep evaluator-owned runner-file injection and every selector command in
   the Runtime backend. Remote Docker's existing workspace fingerprint detects
   the staged local mutation and performs an exact rsync before the next
   container command. Local Docker sees the bind-mounted mutation directly.
5. Preserve all existing cleanup, Runtime resource ownership, timeout,
   evaluation attribution, and public/private artifact boundaries.

The source snapshot materializer remains strict: it never falls back to an
implicit `HEAD`. The legacy projection boundary explicitly selects the logical
base commit when present, or the immutable snapshot repository's concrete HEAD
commit when the imported archive did not preserve the upstream object.

## Alternatives rejected

### Rebuild the frozen images with Git

This would make the existing Profile image identities false or require a new
Profile/Environment identity migration. It cannot close the current frozen
replay gate without invalidating comparability.

### Run Git commands on the Remote host

This depends on undeclared host packages, moves staging outside the declared
container contract, and introduces more Remote command behavior. It is less
portable and less auditable than controller-side staging.

### Reimplement Git patch semantics in Python

The replay requires strict Git-compatible patch behavior, including binary and
mode changes. A partial patch parser would create a new correctness and
security surface.

## Error handling

- Missing or mismatched local workspace authority remains an evaluation
  infrastructure error.
- Source verification timeout maps to `evaluation_timeout`; verification
  failure and identity mismatch retain their existing stable reason codes.
- Agent patch check/apply failure remains `invalid_patch` through
  `StrictPatchApplyError`.
- Hidden patch failure remains `hidden_test_injection_failed`.
- Cleanup always runs for any prepared Runtime lease, including controller
  staging failures.

Private Git stderr may be attached to internal exceptions but must not enter
public results.

## Test strategy

1. Add a focused evaluation test that resolves a gold patch while asserting
   that no Runtime command invokes Git or embeds the old source-hash program.
2. Retain existing strict invalid-patch, source-mismatch, timeout, selector,
   cleanup, Docker, Remote, and full Integrity tests.
3. Prove every v0.5 private source binding selects a locally resolvable commit.
4. Run the focused Runtime suites and full repository suite.
5. Execute one exact Remote baseline replay case that simultaneously exercises
   a synthetic legacy snapshot and controller-side strict staging. Only after
   it matches the frozen expected outcome may a new complete 85-case replay be
   started.

## Safety and compatibility

- No host, port, process, container, or service discovery is added.
- Controller Git operates only on the exact Attempt workspace and never on the
  shared source cache.
- Remote operations remain exact rsync, container execution, and cleanup
  against the configured target.
- Public schemas, Dataset/Task/Source/Environment/Profile identities, v0.5
  artifacts, and scoring rules remain unchanged.
