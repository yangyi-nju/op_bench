Implemented the source-only fix in `torch/nn/modules/module.py`.

The strict unexpected-key check now distinguishes:

- dotted suffixes: must refer to a registered child module
- non-dotted suffixes: must refer to a local parameter/buffer

This prevents keys like `weight.bad_key` or `0.weight.bad_key` from being silently accepted.
