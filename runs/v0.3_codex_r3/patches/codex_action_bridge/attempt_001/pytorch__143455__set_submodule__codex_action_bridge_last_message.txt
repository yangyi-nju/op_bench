Implemented the minimal fix in `torch/nn/modules/module.py`.

`Module.set_submodule()` now handles non-dotted targets as direct child module names via `add_module()`, so names like `"n"` or `"n@1"` are registered as single submodules instead of being misinterpreted by dotted traversal. Dotted paths still traverse parent modules and replace the final child as before.
